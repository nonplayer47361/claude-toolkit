# SDD(Spec-Driven Development) 패턴 디렉토리 템플릿

이 디렉토리는 Spec-Driven Development(SDD) 패턴을 시작하기 위한 템플릿 구조를 제공합니다.

## SDD 패턴 개요
Spec-Driven Development 패턴은 태스크를 수행하기 전, 다음 3개의 문서를 순서대로 작성하고 검증함으로써 작업의 정확성과 효율성을 극대화하는 개발 방식입니다.

1. **요구사항 정의 (`requirements.md`)**: EARS 형식 요구사항 정의
2. **구현 계획 (`plan.md`)**: 구현 세부 계획 및 사용자 승인
3. **검증 체크리스트 (`validation.md`)**: 검증 시나리오 및 자동화 연동

## 사용법
새로운 태스크를 시작할 때, 이 템플릿 디렉토리를 태스크 ID 폴더로 복사하여 사용합니다.
```bash
cp -r references/_specs-template _specs/{TASK_ID}
```

## 각 파일의 역할과 작성 담당자

### 1. `requirements.md` (요구사항 정의)
- **역할**: 태스크에서 해결해야 할 기능적/비기능적 요구사항 및 제약 조건을 명확하게 정의합니다. 요구사항은 EARS(Easy Approach to Requirements Syntax) 형식을 따릅니다.
- **담당자**: 사용자(또는 태스크 지시자)가 기본 요구사항을 제공하며, 에이전트가 이를 상세화하여 보완할 수 있습니다.

### 2. `plan.md` (구현 계획)
- **역할**: 요구사항을 충족하기 위한 구체적인 기술적 접근 방식, 단계별 구현 과정, 위험 요소 및 대안을 작성합니다.
- **담당자**: 에이전트(Claude)가 작성하며, **사용자의 최종 승인**을 받아야 구현을 시작할 수 있습니다.

### 3. `validation.md` (검증 체크리스트)
- **역할**: 구현 결과가 요구사항을 충족하는지 검증하기 위한 자동 및 수동 테스트 체크리스트입니다. `verify.sh` 스크립트와 연동하여 자동 검증을 수행할 수 있도록 합니다.
- **담당자**: 에이전트가 테스트 및 검증 시나리오를 설계하고 작성합니다.
