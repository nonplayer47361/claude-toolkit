# SOUL.md 템플릿

`~/.claude/SOUL.md`에 두는 최상위 운영 지침이다.
가장 먼저 로드되므로 짧고 강한 규칙만 둔다.

## 디스패치 최상위 원칙

1. 매 세션 시작 시 프로젝트 `AGENTS.md`를 먼저 읽고 현재 작업에 적용한다.
2. `PLAN.md`가 없거나 확인되지 않은 상태에서는 dispatch를 시작하지 않는다.
3. 사용자가 계획을 승인한 뒤에만 Phase 1 구현을 시작한다.
4. 같은 오류가 2회 이상 연속되면 반복하지 말고 사람에게 에스컬레이션한다.
5. 컨텍스트 사용량이 40%를 넘으면 `/compact` 후 목표와 남은 작업을 이어간다.

## 에이전트 배분 원칙

- 복잡한 코드 구현, 설계 판단, 테스트 작성, 성능 개선은 `codex` quality 티어에 맡긴다.
- 문서, 주석, TODO, 단순 수정, 반복 정리는 `agy` fast 티어에 맡긴다.
- 인증, 보안, DB, 공개 API, 배포 설정처럼 위험도가 높은 작업은 Cross-Check 프로토콜을 적용한다.
- 배분 전 허용 파일, 금지 파일, 완료 기준을 확인하고 작업 로그에 선택 이유를 남긴다.

## 세션 시작 체크리스트

- [ ] `AGENTS.md`를 읽고 프로젝트 규칙을 확인한다.
- [ ] `.session_state` 또는 Memory MCP에서 이전 맥락을 로드한다.
- [ ] `PLAN.md`를 확인한다.
- [ ] `PLAN.md`가 없으면 Phase 0-A 계획 수립부터 시작한다.
- [ ] 허용 파일과 금지 파일을 확인한다.

## 세션 종료 체크리스트

- [ ] 완료 상태와 남은 작업을 `PLAN.md`에 반영한다.
- [ ] 발견한 패턴, 실패 원인, 중요한 결정을 Memory MCP에 저장한다.
- [ ] `.session_state`를 최신 상태로 갱신한다.
- [ ] 실행한 검증 명령과 결과를 보고한다.
- [ ] 검증하지 못한 항목은 이유와 리스크를 남긴다.

## 금지 행동

- 사용자 승인 없이 `git push`를 실행하지 않는다.
- `verify.sh`가 통과하지 않은 상태를 DONE으로 처리하지 않는다.
- `AGENTS.md`에 명시된 금지 파일을 수정하지 않는다.
- 허용 파일 목록 밖의 파일을 생성하거나 수정하지 않는다.
