# claude-toolkit 개선 및 고도화 로드맵 (Feedback & Next Steps)

본 문서는 `claude-toolkit`의 아키텍처 안정성을 높이고, 이종(異種) 다중 에이전트 환경에서의 자율 루프 효율성을 극대화하기 위한 피드백 및 향후 개선 방향을 정리한 것입니다.

---

## 1. 아키텍처 및 논리적 안정성 강화

### 1.1 서브 에이전트 무한 루프(Looping Trap) 방지 프로토콜
서브 에이전트는 상태를 보존하지 않는 독립 프로세스로 실행되므로, 동일한 에러를 반복할 위험이 큽니다.
* **이슈:** `verify.sh` 실패 시 제공되는 `FEEDBACK.md`에 이전 실패 맥락이 없으면 에이전트가 동일한 코드를 다시 제출할 수 있습니다.
* **해결책:** Phase 5 루프 재배정 시, "직전 시도에서 에이전트가 수행한 행동 요약"과 "실패 원인(Diff)"을 `FEEDBACK.md`에 누적 주입하는 메모리 주입 로직을 추가합니다.

### 1.2 상태 파일(.session_state) 동시성 및 유실 제어
모든 상태가 단일 텍스트 파일에 의존하므로 읽기/쓰기 충돌 시 루프 전체가 붕괴할 수 있습니다.
* **이슈:** `pty-bridge` 데몬과 Claude 오케스트레이터가 동시에 파일에 접근하거나, Rate Limit으로 비정상 종료될 때 파일이 0바이트로 초기화될 위험이 있습니다.
* **해결책:** `update-state.sh` 스크립트에 원자적 쓰기(Atomic Write)를 적용합니다. 임시 파일에 먼저 기록 후 성공 시 원본을 교체(Replace)하며, 항상 `.session_state.bak` 백업본을 유지하도록 변경합니다.

### 1.3 모델 티어 설정 일관성 검토
`dispatch.sh` 내의 에이전트 배정 모델명과 문서 상의 설명 간 불일치를 해소해야 합니다.
* **이슈:** `agy (Antigravity CLI)`는 Gemini 1.5/2.0 기반으로 설명되어 있으나, 티어 테이블에는 `claude-haiku`, `claude-sonnet` 등이 배정되어 있습니다.
* **해결책:** `agy`가 Gemini 기반 CLI라면 코드와 문서 모두 `gemini-2.0-flash` 등으로 명칭을 통일하여 오케스트레이터의 혼동을 방지합니다.

## 2. 성능 최적화 및 비용 절감

### 2.1 동적 컨텍스트 가지치기 (Dynamic Context Pruning)
장기 실행되는 세션에서 오케스트레이터(Claude)의 토큰 소비가 기하급수적으로 증가하는 것을 방지합니다.
* **구현 방안:** 마일스톤 게이트(Phase 4)를 통과할 때마다 전체 세션 대화 로그를 요약본으로 압축하고, 핵심 상태와 다음 작업 목표만을 남기는 스킬 명령어를 추가합니다.

### 2.2 정적 태스크 트리아지 (Static Task Triage)
작업 배정의 효율성을 높이기 위해 단순 통계를 넘어선 사전 분석 라우팅을 도입합니다.
* **구현 방안:** 작업 대상 파일의 확장자, 코드 라인 수, Git Diff 크기 등을 정적으로 사전 분석하여 소형(Codex)과 대형/분석(agy) 중 어느 에이전트가 적합할지 배정 가중치를 자동 계산합니다.

---

## 3. 신규 기능 및 확장 생태계

### 3.1 크로스 에이전트 교차 검증 (Cross-Agent Code Review)
에이전트 간 상호 견제를 통해 코드 품질을 높입니다.
* **구현 방안:** Codex가 구현한 코드를 Git 커밋 및 `verify.sh` 실행 전에, agy 에이전트가 헤드리스 모드로 정적 분석 및 리뷰(`REVIEW.md` 작성)하도록 강제하는 2인 1조 파이프라인을 구축합니다.

### 3.2 실시간 토큰 및 절감 비용 대시보드
멀티 에이전트 시스템 도입으로 인한 비용 절감 효과를 시각화합니다.
* **구현 방안:** 기존 `dashboard.sh`에 모니터링 모듈을 추가하여, 값비싼 Claude 대신 Codex/agy를 사용하여 누적 절감된 추정 비용($)과 각 에이전트의 태스크 성공률을 실시간 터미널 UI로 출력합니다.
