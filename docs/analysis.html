⚡ 30초 요약

claude-toolkit은 Claude Code를 오케스트레이터(지휘자)로 사용하고, Codex CLI(OpenAI)와 agy CLI(Antigravity) 같은 외부 코딩 에이전트를 서브 에이전트로 배정해
코드 구현 → 리뷰 → 4종 자동 검증 → 커밋 흐름을 파일 기반으로 자동화하는 반자동/자동화 멀티 에이전트 개발 워크플로우다.

핵심 가치는 세 가지다: Claude 사용량 최소화(비용 분산), 에이전트별 컨텍스트 격리, 검증된 워크플로우 패턴을 스킬·스크립트로 패키징해 여러 프로젝트에 재사용.

빠른 시작 — 핵심 명령어

# 1. 스킬 설치 (Windows)
.\scripts\install-skill.ps1 -SkillName cli-agent-team

# 2. 프로젝트 초기화 (PLAN.md · AGENT_ROLES.md · .session_state 자동 생성)
bash skills/cli-agent-team/scripts/setup.sh

# 3. CLI 비대화형 실행 가능 여부 검증
bash skills/cli-agent-team/scripts/probe-cli.sh codex limited

# 4. 태스크 배정 (백그라운드 실행 — Claude가 harness로 호출)
bash skills/cli-agent-team/scripts/dispatch.sh codex T001 limited . execute

# 5. 완료 검증 (4종 자동 검사)
bash skills/cli-agent-team/scripts/verify.sh T001

🎯 프로젝트 의도

claude-toolkit은 Claude Code(Anthropic)를 단순 코딩 보조 도구가 아니라 오케스트레이터(지휘자)로 격상시켜, 외부 CLI 기반 AI 에이전트를 부하 에이전트로 부리는 이종(異種) 멀티 에이전트 개발 루프를 구축하고 여러 프로젝트에 배포하는 작업 공간이다.

① 비용 분산

Claude 사용량은 비용 부담이 커질 수 있으므로, 구현 작업은 외부 CLI 에이전트에 위임하고 Claude는 계획·검토·승인·커밋에만 집중한다.

② 컨텍스트 격리

단일 Claude 세션에서 긴 작업을 하면 컨텍스트 오염 가능성이 높아진다. 에이전트별 독립 프로세스로 이를 최소화한다.

③ 재사용 패키징

검증된 워크플로우 패턴·스크립트·문서를 Claude Code 스킬/에이전트/MCP 서버 형식으로 묶어 팀 전체에 배포한다.

💡
핵심 철학: 기본 오케스트레이션은 MCP 서버 없이 Bash + 스킬만으로 처리한다. 외부 CLI 호출은 codex exec, agy --print 명령 하나면 충분하다. 단, agy의 headless stdout 문제처럼 CLI 실행 환경 자체를 보정해야 하는 예외 상황에서는 pty-bridge 같은 보조 래퍼를 사용할 수 있다. 진짜 재사용 가치는 코드가 아니라 워크플로우 지식(파일 구조, 정확한 플래그, 검증된 함정 목록)에 있다.

누가 만들었나

Antigravity IDE 사용자가 실제 프로젝트(Tarkov Market KO)에서 Codex CLI + agy를 직접 자동화하며 발견한 패턴을 범용 스킬로 패키징했다. 토큰 비용에 민감하고 멀티 에이전트 오케스트레이션을 실전에서 검증하려는 개인 개발자.

스킬 설계 원칙

skills/<이름>/SKILL.md — 프론트매터 name·description 필수. description이 system-reminder에 그대로 표시되므로 트리거 상황을 명확히 기술해야 한다.

에이전트 설계 원칙

agents/<이름>.md — name·description·tools 필수. "프로젝트 컨텍스트" 섹션에 담당 파일 경로를 명시하는 것이 에이전트 품질을 결정한다.

MCP 서버 설계 원칙

mcp-servers/<이름>/ — Node.js ESM, stdio transport. @modelcontextprotocol/sdk 기반. register-mcp.ps1으로 settings.json에 자동 등록.

상태 관리 원칙

_agent_reports/.session_state가 루프 상태의 단일 출처. rate limit으로 Claude가 끊겨도 이 파일 하나를 읽으면 즉시 재개 가능. 원자적 쓰기로 보호해야 파일 손상 위험을 줄일 수 있다.

⚙️ 핵심 스킬: cli-agent-team

현재 이 저장소의 유일한 완성 스킬. Claude를 오케스트레이터로, Codex CLI(OpenAI)와 Antigravity CLI/agy를 서브 에이전트로 부리는 반자동/자동화 개발 루프를 프로젝트에 구성한다.

🧰 활용 도구 · MCP · 스킬

외부 CLI 에이전트 (서브 에이전트)

⚠️
agy 모델 주의: agent-characteristics.md에서 agy를 "Google Gemini 기반"으로 기술하고 있으나, dispatch.sh의 --model 플래그에는 현재 Anthropic 계열 모델 ID가 기재되어 있다. agy가 내부적으로 여러 백엔드를 지원하거나 설정값이 업데이트 중일 가능성이 있으므로, 실제 사용 전 자신의 agy 버전과 계정에서 지원하는 모델 ID를 직접 확인해야 한다. 아래 표의 모델명은 코드 내 설정값의 예시이며 확정된 공식 명칭이 아니다.
에이전트별 특성 비교 — rate limit·컨텍스트 수치는 공급사 정책에 따라 변동됨
에이전트실행 경로컨텍스트강점최적 작업
Codex CLI OpenAI 계열 코드 모델 호출 대형 정밀 코드 생성, 구조적 리팩토링 소·중형 구현 (파일 2~5개)
agy Antigravity CLI (백엔드 모델은 설정에 따라 다름) 최대 1M tokens 대형 코드베이스 전체 분석, 광범위 리팩토링 대형 구현·분석·문서화
⚠️
agy 알려진 버그: 헤드리스 모드(TTY 없이 실행)에서 stdout에 채팅 텍스트가 안 보인다 (tty 미감지로 인한 출력 억제로 추정). 그러나 실제 파일 쓰기 작업은 정상 수행된다. 이를 우회하기 위해 mcp-servers/pty-bridge/에 node-pty ConPTY 래퍼를 구현했다. stdout이 비어 있어도 verify.sh의 4번 증거 파일 검사로 완료를 판정한다.

dispatch.sh 지원 모델 티어

아래 모델 ID는 dispatch.sh 내 현재 설정값 예시다. 실제 사용 가능한 모델명은 CLI 버전과 계정 권한에 따라 달라질 수 있다.
티어Codex 설정값 (예시)agy 설정값 (예시)용도
fast OpenAI 소형 고속 모델 Anthropic 소형 고속 모델 문서·주석·단순 수정
quality OpenAI 고성능 코드 모델 Anthropic 고성능 모델 코드 구현·인증·보안 (기본값)
🔴
agy 모델 일관성 이슈: dispatch.sh에 현재 Anthropic 모델 ID가 agy 설정값으로 기재되어 있으나, agy는 Antigravity CLI(Google 기반)다. 이 값이 실제로 작동하는지, 아니면 수정이 필요한지 E2E 파이프라인 테스트 완료 전까지 확정할 수 없다. 실제 사용 시 probe-cli.sh로 먼저 검증할 것.

node-pty ConPTY 래퍼. agy headless stdout 침묵 문제 우회용. mcp-servers/pty-bridge/에 위치.

⚖️ 장점 · 단점 · 한계

✅ 장점

💰
토큰 비용 최소화

Claude는 오케스트레이터 역할(계획·검토·커밋)에만 집중. 오타·1~3줄 수정도 재배정해 Claude 사용량 낭비 없음.

🧩
이종 에이전트 조합

서로 다른 공급사(OpenAI + Antigravity)를 조합. rate limit pool이 독립적이므로 하나가 한도 초과해도 다른 에이전트로 폴백 가능.

🔄
rate limit 내성

SESSION_STATE + ScheduleWakeup 이중 안전망. Claude rate limit으로 루프가 끊겨도 정확한 단계에서 자동 재개.

🎯
실전 검증 패턴 (일부)

개별 CLI 호출, 파일 프로토콜, 실전 태스크 배정은 Tarkov Market KO에서 직접 검증. 이론이 아닌 실측 데이터 기반의 알려진 함정 목록 보유.

📊
적응형 배분 학습

window 알고리즘과 task_type별 성능 점수로 배정이 세션이 쌓일수록 자동 최적화. 단순 고정 비율이 아닌 데이터 기반 라우팅.

🛡️
엄격한 권한 관리

완전자율 플래그를 프로젝트마다 명시적으로 확인. 한 프로젝트의 승인이 다른 프로젝트에 전이되지 않음.

❌ 단점 · 한계

🔬
E2E 검증 미완성

전체 자동 루프의 end-to-end 안정성 검증은 아직 진행 중이다. 일부 실전 태스크에서 검증됐지만, dispatch → pty-bridge → agy → REPORT → verify 전체 파이프라인은 추가 검증이 필요하다.

🔀
agy 모델 설정 불일치

agent-characteristics.md는 agy를 "Google Gemini 기반"으로 기술하나, dispatch.sh에는 Anthropic 계열 모델 ID가 기재돼 있다. 실제 사용 전 probe-cli.sh로 직접 검증이 필요하다.

⚡
에이전트 반복 실패 위험

서브 에이전트는 Stateless 독립 프로세스라 "직전 세션의 실패 이유"를 기억하지 못한다. FEEDBACK.md에 히스토리가 누적되지 않으면 같은 실수를 반복할 수 있다.

📄
.session_state 동시성 위험

단일 텍스트 파일을 여러 프로세스가 동시에 읽고 쓸 경우 파일이 0바이트로 깨질 수 있다. update-state.sh의 원자적 쓰기 보호가 완전히 구현되지 않으면 루프 전체가 무너질 수 있다.

🖥️
Windows 환경 복잡성

pexpect/winpty 미지원, Git Bash 한글 경로 문제, 백슬래시 따옴표 이슈 등 Windows 특유의 함정이 많다. 이를 모두 문서화하고 우회책을 구현했지만 진입 장벽이 있다.

📦
1개 스킬만 완성

현재 cli-agent-team만 완성 상태. git-helper, code-review-ko 등은 개발 예정. MCP 서버도 템플릿 + pty-bridge 이외에는 구현된 서버 없음.

🛠️ 한계 대응 전략

agy stdout 침묵 → pty-bridge

mcp-servers/pty-bridge/run.js — node-pty의 ConPTY를 사용해 agy를 가상 TTY로 실행, stdout을 로그 파일로 캡처. dispatch.sh의 agy 케이스가 이 래퍼를 자동으로 경유한다.

rate limit 중단 → 이중 안전망

① 루프 시작 시 "리셋주기+300초" 장기 ScheduleWakeup 예약. ② 정상 완료 시 10초 즉시 재예약으로 덮어쓰기. Claude가 언제 끊겨도 .session_state를 읽어 재개.

에이전트 리밋 → 폴백 체인

소·중형: Codex → agy → Claude(최후). 대형·분석: agy → Codex(분할) → Claude(최후). 3회 피드백 미해결 시 BLOCKED 자동 처리 후 루프 유지. BLOCKED 항목은 마일스톤 게이트에서 사용자에게 일괄 보고.

에이전트 반복 실패 → 히스토리 FEEDBACK

FEEDBACK.md 재작성 시 "직전 시도 요약 + 실패 원인 + 이번에 다른 접근"을 회차별로 누적 기록. 단순 에러 메시지만 전달하면 에이전트가 같은 코드를 재제출할 수 있다.

.session_state 손상 → 원자적 쓰기

쓰기 전 백업본(.session_state.bak) 생성 후 임시 파일에 쓰고 성공 시 rename하는 방식으로 update-state.sh를 보호해야 한다. 현재 개선이 필요한 항목으로 기록됨.

dispatch.sh exit 오판 → EXIT trap

에이전트가 내부 검증 중 non-zero exit를 내도 REPORT.md에 완료 AC(- [x])가 있으면 성공으로 처리하는 _on_exit trap이 dispatch.sh에 내장.
