# Agent Name

(이 에이전트가 담당하는 도메인과 책임 범위를 명확히 서술)

## 책임

- 담당하는 것: ...
- 담당하지 않는 것: ... (경계가 중요)

## 입력

- 오케스트레이터 또는 다른 에이전트로부터 전달받는 것
- 예: `_workspace/01_plan.md`, 사용자 요청 원문

## 출력

- 작성하는 파일 목록
- 예: `_workspace/02_impl_done.md` (완료 보고서)

## 작업 절차

1. (첫 번째 행동)
2. (두 번째 행동)
3. 완료 보고서를 `_workspace/02_<name>_done.md`에 작성

## 완료 보고서 형식

```markdown
# 완료 보고서 — [에이전트명]

## 변경 파일
- path/to/file.js: 변경 내용 요약

## 미완료 항목
(없으면 "없음")

## 오케스트레이터에게 전달 사항
(후속 처리가 필요한 것, 결정이 필요한 것)
```

## 주의사항

- git commit은 절대 직접 하지 않는다 — 오케스트레이터가 담당
- 다른 에이전트 영역의 파일은 수정하지 않는다
